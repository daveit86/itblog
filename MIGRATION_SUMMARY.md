## Next Steps

### Option 1: Quick Deploy to Vercel (Recommended)

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Create a Vercel account and project**

3. **Add Vercel Postgres** (in Vercel Dashboard → Storage → Create Database)

4. **Configure environment variables** in Vercel:
   - `DATABASE_URL` (auto-generated by Vercel Postgres)
   - `NEXTAUTH_SECRET` (generate: `openssl rand -base64 32`)
   - `NEXTAUTH_URL` (your domain)
   - `ADMIN_EMAIL` and `ADMIN_PASSWORD`

5. **Deploy:**
   ```bash
   vercel --prod
   ```

6. **Run migrations** (after first deploy):
   ```bash
   npx prisma migrate deploy
   # Or use Vercel CLI:
   vercel --prod
   ```

7. **Seed database** (optional, creates admin user):
   ```bash
   npx ts-node prisma/seed.ts
   ```

### Option 2: Local Development with PostgreSQL

1. **Install PostgreSQL** locally or use Docker:
   ```bash
   docker run -d --name itblog-db \
     -e POSTGRES_PASSWORD=password \
     -e POSTGRES_DB=itblog \
     -p 5432:5432 postgres:15
   ```

2. **Create `.env` file:**
   ```bash
   cp .env.example .env
   ```

3. **Edit `.env`:**
   ```
   DATABASE_URL="postgresql://postgres:password@localhost:5432/itblog"
   NEXTAUTH_SECRET="your-secret-here"
   NEXTAUTH_URL="http://localhost:3000"
   ADMIN_EMAIL="admin@example.com"
   ADMIN_PASSWORD="admin123"
   ```

4. **Run migrations:**
   ```bash
   npm run db:migrate
   ```

5. **Seed database:**
   ```bash
   npm run db:seed
   ```

6. **Start development server:**
   ```bash
   npm run dev
   ```

## Troubleshooting

### "Can't reach database server"
- Make sure PostgreSQL is running
- Check your `DATABASE_URL` format: `postgresql://user:pass@host:port/dbname`

### "Prisma Client initialization error"
- Run `npm run postinstall` to regenerate the client
- Check that `DATABASE_URL` is set correctly

### Migration errors
- For Vercel: Use `npx prisma migrate deploy` (not `migrate dev`)
- Locally: Use `npm run db:migrate`
